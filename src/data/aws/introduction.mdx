<ul className="list-arrow">
  <li>An <strong>AWS account</strong> with Bedrock model access enabled — <a className="brand-link" href="https://console.aws.amazon.com/bedrock" target="_blank">console.aws.amazon.com/bedrock</a></li>

## Regions and Inference Profiles

Many Amazon foundation models require an **inference profile** rather than direct on-demand invocation. Use the system-defined cross-region prefixes:

<div className="flex gap-4 items-start">
  <div className="w-1/2 min-w-0">
    <pre className="overflow-x-auto"><code className="language-python">{`import boto3

# Direct model IDs still work for some models
MODEL_ID = "amazon.nova-pro-v1:0"

# Cross-region inference profiles (required for Nova and others)
EU_MODEL  = "eu.amazon.nova-micro-v1:0"   # routes across EU regions
US_MODEL  = "us.amazon.nova-pro-v1:0"     # routes across US regions

bedrock = boto3.client(
    service_name='bedrock-runtime',
    region_name='us-east-1',
)`}</code></pre>
  </div>
  <div className="w-1/2 min-w-0">
    <ul className="list-arrow">
      <li>Enable model access in <strong>AWS Console → Bedrock → Model access</strong> before first use</li>
      <li>If you get <em>"on-demand throughput isn't supported"</em>, switch to the cross-region inference profile ID (e.g. <code>eu.amazon.nova-micro-v1:0</code>)</li>
      <li>Guardrail and model must be in the <strong>same region</strong> when using <code>invoke_model</code></li>
    </ul>
  </div>
</div>
