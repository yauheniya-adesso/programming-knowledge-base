It provides the building blocks to design, build, and operate **Agentic AI** applications at scale.

An **Agentic AI** application is one where an AI model doesn't just respond to a single prompt — it autonomously plans, uses tools, retrieves context, and takes multi-step actions to complete complex goals. Azure provides a managed, production-ready stack for every layer of these apps.

## Prerequisites

Before building, you'll need:

<ul className="list-arrow">
  <li>An <strong>Azure subscription</strong> — <a className="brand-link" href="https://portal.azure.com" target="_blank">portal.azure.com</a></li>
  <li>The <strong>Azure CLI</strong> installed: <code>brew install azure-cli</code> (macOS) or <a className="brand-link" href="https://learn.microsoft.com/en-us/cli/azure/install-azure-cli" target="_blank">docs</a></li>

  <li>Required Python packages (install per section below)</li>

## Environment Variables Pattern

All Azure services are connected via environment variables. Follow this pattern throughout the tutorials:

<pre className="inline-block"><code className="language-bash">{`# .env
AZURE_OPENAI_ENDPOINT=https://<resource>.openai.azure.com/
AZURE_OPENAI_API_KEY=<key>
AZURE_OPENAI_DEPLOYMENT=<deployment-name>

AZURE_SEARCH_ENDPOINT=https://<resource>.search.windows.net
AZURE_SEARCH_KEY=<admin-key>
AZURE_SEARCH_INDEX=<index-name>

AZURE_STORAGE_CONNECTION_STRING=DefaultEndpointsProtocol=https;...
AZURE_STORAGE_CONTAINER=<container-name>

AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT=https://<resource>.cognitiveservices.azure.com/
AZURE_DOCUMENT_INTELLIGENCE_KEY=<key>

# For SQL / Entra ID auth
AZURE_CLIENT_ID=<app-client-id>
AZURE_CLIENT_SECRET=<client-secret>
AZURE_TENANT_ID=<tenant-id>`}</code></pre>

<ul className="list-arrow">
  <li>Never commit <code>.env</code> to source control — add it to <code>.gitignore</code></li>
  <li>In production (Azure DevOps pipelines / App Service), inject these as <strong>pipeline variables</strong> or <strong>Key Vault secrets</strong></li>
</ul>
