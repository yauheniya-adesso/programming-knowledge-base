A *coding agent* is a language model that doesn't just answer — it **acts**, by calling tools in a loop until a task is done. Anthropic's published guidance defines an agent as a system where "LLMs dynamically direct their own processes and tool usage, maintaining control over how they accomplish tasks" (Anthropic Engineering, *Building Effective Agents*, December 2024).

---

## The Agentic Loop

Under the hood, Claude Code runs a loop that looks roughly like this:

<pre className="inline-block"><code className="language-text">{`  ┌─────────────────────────────────────────────────┐
  │  1. User sends a task                           │
  │              ↓                                  │
  │  2. Model thinks, then either:                  │
  │       • produces a final answer  →  done        │
  │       • emits a tool call                       │
  │              ↓                                  │
  │  3. Harness runs the tool (Read, Bash, Edit…)   │
  │              ↓                                  │
  │  4. Tool result is appended to the context      │
  │              ↓                                  │
  │  5. Loop back to step 2                         │
  └─────────────────────────────────────────────────┘`}</code></pre>

The model never executes anything itself — it only **requests** a tool call. The Claude Code harness runs the tool on your machine (subject to permissions) and returns the output as the next message. This is the same **tool use** pattern documented in the Claude API.

## What Makes the Loop Work

Three properties from Anthropic's research and documentation are worth knowing:

<div className="flex gap-4 items-start">
  <div className="w-1/3 min-w-0">
    <p><Icon icon="lucide:anchor" height={32} style={{display: 'inline', verticalAlign: 'middle', color: 'var(--color-blue)', marginRight: '0.75rem'}} /> <strong>Grounded in reality</strong></p>
    <p>Each tool result is real output from your machine — not a hallucination. The model gets to see what actually happened (errors, stack traces, test results) and react.</p>
  </div>
  <div className="w-1/3 min-w-0">
    <p><Icon icon="lucide:refresh-cw" height={32} style={{display: 'inline', verticalAlign: 'middle', color: 'var(--color-blue)', marginRight: '0.75rem'}} /> <strong>Self-correcting</strong></p>
    <p>If a command fails, the failure becomes the next observation. The model can read the error, adjust the plan, and try again — the loop is the error-recovery mechanism.</p>
  </div>
  <div className="w-1/3 min-w-0">
    <p><Icon icon="lucide:gauge" height={32} style={{display: 'inline', verticalAlign: 'middle', color: 'var(--color-blue)', marginRight: '0.75rem'}} /> <strong>Bounded by context</strong></p>
    <p>The whole loop happens inside the model's context window. Long tasks compress old messages, summarise tool output, or hand off to subagents to stay under the limit.</p>
  </div>
</div>
