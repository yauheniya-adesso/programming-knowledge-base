          <li><strong>Harness</strong> — the program around the model (the Claude Code CLI / VS Code extension). It receives the tool call, actually runs it on your machine, and enforces permissions.</li>
          <li><strong>Next observation</strong> — the tool's result (file contents, command output, error message) appended to the conversation as the next message the model reads before deciding the next step.</li>
        </ul>
      </li>

## Models You Can Use

Claude Code lets you switch between Claude models for different trade-offs of capability, speed, and cost. Switch with <code>/model</code> at any time.

<table className="brand-table">
  <thead>
    <tr>
      <th>Model</th>
      <th>Strength</th>
      <th>When to pick it</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Opus 4.7</strong></td>
      <td>Highest reasoning, supports a 1M-token context window</td>
      <td>Long, complex changes spanning many files</td>
    </tr>
    <tr>
      <td><strong>Sonnet 4.6</strong></td>
      <td>Balanced quality and speed</td>
      <td>Default for most day-to-day work</td>
    </tr>
    <tr>
      <td><strong>Haiku 4.5</strong></td>
      <td>Fastest and cheapest</td>
      <td>Quick edits, simple questions, throwaway scripts</td>
    </tr>
  </tbody>
</table>

<ul className="list-arrow">
  <li>The <strong>knowledge cutoff</strong> for Claude 4.x is January 2026 — for newer libraries, point Claude at the docs</li>
  <li>Long sessions auto-compress context once it gets close to the model's limit</li>
</ul>
