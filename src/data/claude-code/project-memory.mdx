Claude Code reads <code>CLAUDE.md</code> files automatically at the start of every session. They are how you give Claude **lasting, project-specific context** — coding conventions, build commands, gotchas — so you don't have to repeat yourself in every prompt.

All three are merged into Claude's context. Project rules override user rules when they conflict.

## A Good <code>CLAUDE.md</code>

Keep it short, factual, and high-signal. Claude doesn't need prose — it needs *rules and pointers*.

<pre className="inline-block"><code className="language-markdown">{`# Build & Test
- Install: \`pnpm install\`
- Dev server: \`pnpm dev\`
- Run tests: \`pnpm test\` (NOT \`npm test\` — wrong lockfile)
- Type-check before committing: \`pnpm typecheck\`

# Conventions
- TypeScript strict mode — no \`any\`
- React components: function components + hooks only
- Tailwind for styling, no CSS modules

# Architecture
- API routes live in \`src/server/routes/\`
- Shared types in \`src/types/\` — keep them framework-agnostic

# Don't
- Don't run \`pnpm install\` without asking
- Don't edit files in \`generated/\` — they come from codegen`}</code></pre>

## Adding to Memory From the Chat

The fastest way to update <code>CLAUDE.md</code> is to start a message with <code>#</code>:

<pre className="inline-block"><code className="language-bash">{`# All new components go in src/ui/
# and follow the existing PascalCase file naming.`}</code></pre>

Claude will:

<ol className="list-numbered">
  <li>Detect the <code>#</code> prefix</li>
  <li>Ask whether to save to project, local, or user memory</li>
  <li>Append the rule and confirm</li>
</ol>

<ul className="list-arrow">
  <li>Use <code>/init</code> in a new project to generate a starter <code>CLAUDE.md</code></li>
  <li>Review your <code>CLAUDE.md</code> regularly — stale rules lead Claude astray</li>
  <li>If a rule keeps getting ignored, it's usually too vague — make it concrete and add the <em>why</em></li>
</ul>
