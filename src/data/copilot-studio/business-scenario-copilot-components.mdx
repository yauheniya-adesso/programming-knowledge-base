## Instructions

### Step 1 — Define the Triggers

Write **5–7 natural-language phrases** users might say to begin this interaction.

<ul className="list-arrow">
  <li>"Request vacation"</li>
  <li>"How many days off do I have left?"</li>
  <li>"I want to take time off next week"</li>
  <li>"Submit a leave request"</li>
  <li>"Check my leave balance"</li>
  <li>"When will my vacation be approved?"</li>
  <li>"Cancel my time-off request"</li>
</ul>

---

### Step 2 — Outline the Topics

Break the interaction into **2–3 reusable topics**. For each topic, sketch the logic flow:

<div className="grid grid-cols-1 gap-4 my-4">

  <div className="mode-pill-card">
    <div className="mode-pill-card-title">
      <Icon icon="fluent:calendar-checkmark-24-regular" height={20} />
      <strong>Topic 1 — Check Leave Balance</strong>
    </div>
    <ul className="list-arrow m-0">
      <li><strong>Bot asks:</strong> "Which type of leave would you like to check? (Annual / Sick / Unpaid)"</li>
      <li><strong>Condition:</strong> If employee ID is not on record, prompt for employee ID</li>
      <li><strong>Action:</strong> Call Power Automate flow → query HRIS for balance</li>
      <li><strong>Response:</strong> Return remaining days by leave type</li>
    </ul>
  </div>

  <div className="mode-pill-card">
    <div className="mode-pill-card-title">
      <Icon icon="fluent:edit-24-regular" height={20} />
      <strong>Topic 2 — Submit a Time-Off Request</strong>
    </div>
    <ul className="list-arrow m-0">
      <li><strong>Bot asks:</strong> Start date, end date, leave type, optional reason</li>
      <li><strong>Condition:</strong> Check if requested days ≤ available balance</li>
      <li><strong>Branch:</strong> If insufficient balance → inform user and suggest alternatives</li>
      <li><strong>Action:</strong> Trigger Power Automate flow → submit request to approval system (e.g., SharePoint list / HRIS)</li>
      <li><strong>Response:</strong> Confirmation message with request ID and expected approval timeline</li>
    </ul>
  </div>

  <div className="mode-pill-card">
    <div className="mode-pill-card-title">
      <Icon icon="fluent:dismiss-circle-24-regular" height={20} />
      <strong>Topic 3 — Cancel a Leave Request</strong>
    </div>
    <ul className="list-arrow m-0">
      <li><strong>Bot asks:</strong> Request ID or date range of the leave to cancel</li>
      <li><strong>Condition:</strong> Check if request is still in "Pending" status</li>
      <li><strong>Branch:</strong> If already approved → escalate to HR or provide manual instructions</li>
      <li><strong>Action:</strong> Power Automate flow → update request status in HRIS</li>
      <li><strong>Response:</strong> Cancellation confirmation or escalation notice</li>
    </ul>
  </div>

</div>

---

### Step 3 — Add Generative Fallbacks

Consider open-ended questions a user might ask **outside** of defined topics.

<div className="mode-pill-card">
  <p><strong>Example question:</strong> <em>"Can I take half-days on Fridays in August?"</em></p>
  <p className="m-0"><strong>Fallback strategy:</strong></p>
  <ul className="list-arrow m-0">
    <li>The bot uses a <strong>knowledge source</strong> (e.g., HR policy document in SharePoint) to generate a grounded answer via generative AI.</li>
    <li>If no relevant content is found → the bot acknowledges it cannot answer with certainty and <strong>escalates to a human HR agent</strong> via handoff.</li>
    <li>Generative answers are clearly labeled as AI-generated and link back to the source policy document.</li>
  </ul>
</div>

---

### Step 4 — Document the Flow

```
User Input
    │
    ▼
[Trigger Matching]
    ├── "check balance"      → Topic: Check Leave Balance
    │                              └── Power Automate → HRIS query
    │
    ├── "submit / request"   → Topic: Submit Time-Off Request
    │                              └── Power Automate → HRIS / SharePoint write
    │
    ├── "cancel"             → Topic: Cancel Leave Request
    │                              └── Power Automate → HRIS update
    │
    └── No match             → Generative Fallback
                                   ├── Knowledge source (SharePoint HR Docs)
                                   └── Escalate to HR agent if unresolved
```

**Tool integrations:**
<ul className="list-arrow">
  <li><strong>SharePoint</strong> — HR policy knowledge base + leave request list</li>
  <li><strong>HRIS</strong> (e.g., SAP, Workday) — leave balances and approvals via Power Automate connector</li>
  <li><strong>Power Automate</strong> — orchestrates all backend actions</li>
</ul>
