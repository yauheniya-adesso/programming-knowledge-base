The core building blocks of every Copilot Studio agent — understand these and you can design any conversational workflow.

## Anatomy of an Agent

Every Copilot Studio agent is made of four layers that work together:

## Topics

A **topic** is the fundamental unit of conversation logic. Each topic:

<div className="grid grid-cols-2 gap-4 my-4">

  <div className="mode-pill-card">
    <div className="mode-pill-card-title">
      <Icon icon="fluent:comment-multiple-24-regular" height={22} />
      <strong>Trigger Phrases</strong>
    </div>
    <p className="text-sm">3–10 example phrases that activate the topic. The NLU model matches user input against these phrases to decide which topic fires.</p>
    <p className="text-sm m-0"><em>Example: "How many vacation days do I have?", "Check my leave balance"</em></p>
  </div>

  <div className="mode-pill-card">
    <div className="mode-pill-card-title">
      <Icon icon="fluent:branch-fork-24-regular" height={22} />
      <strong>Conversation Nodes</strong>
    </div>
    <ul className="list-arrow m-0 text-sm">
      <li><strong>Message</strong> — bot sends text, cards, images</li>
      <li><strong>Question</strong> — bot asks &amp; captures input into a variable</li>
      <li><strong>Condition</strong> — branch on variable values</li>
      <li><strong>Action</strong> — call a Power Automate flow</li>
      <li><strong>Redirect</strong> — hand off to another topic</li>
      <li><strong>End conversation</strong> — close or escalate</li>
    </ul>
  </div>

</div>

<div className="border-l-4 border-blue pl-4 py-1 my-4 text-sm">
  <strong>System topics</strong> are built-in topics (Greeting, Escalate, Fallback, End of Conversation) that handle universal conversation moments. You can customize but not delete them.
</div>

## Entities & Variables

<table className="brand-table">
  <thead>
    <tr>
      <th>Concept</th>
      <th>Description</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Entity</strong></td>
      <td>A typed data extractor — e.g. <code>Number</code>, <code>Date</code>, <code>Email</code>, custom lists. Used in Question nodes to validate and parse user input automatically.</td>
    </tr>
    <tr>
      <td><strong>Variable</strong></td>
      <td>A named slot that stores a value during a conversation. Can be <code>topic</code> (local) or <code>global</code> (shared across topics).</td>
    </tr>
    <tr>
      <td><strong>System variables</strong></td>
      <td>Pre-built variables like <code>System.User.DisplayName</code>, <code>System.LastMessage.Text</code>, <code>System.Activity.Channel</code>.</td>
    </tr>
  </tbody>
</table>

## Actions (Power Automate Flows)

<div className="mode-pill-card">
  <div className="mode-pill-card-title">
    <Icon icon="fluent:plug-connected-24-regular" height={22} />
    <strong>Actions connect your bot to real systems</strong>
  </div>
  <ul className="list-arrow m-0">
    <li>An <strong>Action node</strong> calls a Power Automate cloud flow synchronously.</li>
    <li>The flow can query <strong>Dataverse</strong>, call <strong>REST APIs</strong>, send emails, update SharePoint lists, or write to any connector.</li>
    <li>Input and output parameters bind directly to topic variables.</li>
    <li>You can also call <strong>HTTP requests</strong> directly (without a flow) using the HTTP action node (preview).</li>
  </ul>
</div>

## Quick Mental Model

```
User message
    │
    ▼
NLU / Generative Orchestrator
    ├── Matches topic trigger  →  Topic flow
    │                               ├── Questions (collect variables)
    │                               ├── Conditions (branch)
    │                               ├── Actions (call flows / APIs)
    │                               └── Messages (respond)
    │
    └── No topic match        →  Generative Answers (knowledge sources)
                                        └── Still no answer → Escalate / Fallback topic
```
