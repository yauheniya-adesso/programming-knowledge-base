Basic git terms to manage repositories and collaborate efficiently:

  • SSH protocol – A secure network protocol used to authenticate and transfer data between a local machine and a remote repository.
    ssh -T git@github.com
  • Repository – A storage location for all the files, history, and changes of a project in Git.
    git init project-name
  • Forking – Forking creates a copy of a GitHub repository on which one can work without affecting the original repository. After installing GitHub CLI (gh) on your system, e.g., brew install gh (on macOS):
    gh auth login --web
    gh repo fork https://github.com/ORIGINAL-OWNER/REPO-NAME
  • Cloning – A process of creating a copy of the project's code and its complete version history from the remote repository on the local machine.
    git clone https://github.com/USERNAME/REPO-NAME.git
  • Upstream – A term used by developers to refer to the original source where the local copy was cloned from.
    git remote add upstream https://github.com/ORIGINAL-OWNER/REPO-NAME.git
  • Working directory – The local folder on your computer where files from a Git repository are checked out and edited.
    pwd
  • Branch – A separate line of development within a repository that allows multiple versions of a project to exist simultaneously.
    git branch feature-branch
  • Staging area – An area where commits can be formatted and reviewed before completing the commit.
    git add -p
  • Config – Git configuration settings that store user information, preferences, and repository-specific options across local, global, or system scopes.
    git config --local user.name "Your Name"
  • Commit – A snapshot of changes in the repository, along with a descriptive message of what was changed.
    git commit -m "category: Describe your changes in present tense and active voice"
  • Merge – The process of combining changes from different branches into a single branch.
    git merge feature-branch
  • Pull request – A request to merge changes from one branch or fork into another branch, typically in the original repository.
    gh pr create --repo USERNAME/REPO-NAME
