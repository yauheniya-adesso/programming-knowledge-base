Example Git workflow for creating, merging, and deploying a patch:

  • Clone repository – Create a local copy of the remote repository including its full history.
    git clone https://github.com/repository/project.git
  • Create patch branch – Create a new branch for isolated patch development.
    git branch PATCH-1
  • Switch branch – Move from the current branch to the patch branch.
    git checkout PATCH-1
    Alternative (create and switch in one step):
    git checkout -b PATCH-1
  • Check repository state – View current branch, modified files, and staging status.
    git status
  • Review changes – Show differences between the working directory and the last commit.
    git diff
  • Stage changes – Add modified files to the staging area in preparation for a commit.
    git add .
  • Unstage changes – Remove a file from the staging area before committing.
    git restore --staged <file>
  • Commit patch – Save a snapshot of staged changes with a descriptive message.
    git commit -m "fix: Execute patch"
  • Return to main branch – Switch back to the stable branch.
    git checkout main
  • Merge patch – Integrate the patch branch into the main branch.
    git merge PATCH-1
  • Delete patch branch – Remove the branch after successful merge.
    git branch -d PATCH-1
  • Push main branch – Upload local commits and set upstream tracking for future pushes.
    git push -u origin main
  • Stage version update – Prepare version or metadata changes for release.
    git add .
  • Commit release update – Commit housekeeping or version bump changes.
    git commit -m "chore: Update package version"
  • Push final changes – Upload the latest commits to the remote repository.
    git push
