Abstract classes cannot be instantiated and may contain abstract methods (methods without implementation).

Defining Abstract Class:

public abstract class AIModel {
    protected String name;
    protected String provider;
    protected int releaseYear;
    
    // Regular method with implementation
    public void displayInfo() {
        System.out.println(name + " by " + provider + " (" + releaseYear + ")");
    }
    
    // Abstract method (no implementation)
    public abstract void generate(String prompt);
    public abstract void shutdown();
}

Implementing Abstract Class:

public class ChatModel extends AIModel {
    public ChatModel(String name, String provider, int releaseYear) {
        this.name = name;
        this.provider = provider;
        this.releaseYear = releaseYear;
    }
    
    // Must implement all abstract methods
    @Override
    public void generate(String prompt) {
        System.out.println(name + " is generating a chat response for: " + prompt);
    }
    
    @Override
    public void shutdown() {
        System.out.println(name + " chat model is shutting down");
    }
}

public class ImageModel extends AIModel {
    public ImageModel(String name, String provider, int releaseYear) {
        this.name = name;
        this.provider = provider;
        this.releaseYear = releaseYear;
    }
    
    @Override
    public void generate(String prompt) {
        System.out.println(name + " is generating an image for: " + prompt);
    }
    
    @Override
    public void shutdown() {
        System.out.println(name + " image model is shutting down");
    }
}

Using Abstract Classes:

public class Main {
    public static void main(String[] args) {
        // Cannot instantiate abstract class
        // AIModel m = new AIModel(); // ERROR
        
        AIModel model1 = new ChatModel("Claude Sonnet 4.6", "Anthropic", 2025);
        AIModel model2 = new ImageModel("DALL-E 3", "OpenAI", 2024);
        
        model1.displayInfo();
        model1.generate("Explain polymorphism");
        model1.shutdown();
        
        System.out.println();
        
        model2.displayInfo();
        model2.generate("A sunset over mountains");
        model2.shutdown();
    }
}

When to Use Abstract Classes:

  • When you want to share code among related classes
  • When you need to define common behavior with some methods implemented
  • When you want to declare non-static, non-final fields
  • When you need access modifiers other than public
