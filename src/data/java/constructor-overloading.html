Constructor overloading allows a class to have multiple constructors with the same name but different parameters (different number or type). It is similar to method overloading, but constructors are special methods called automatically when an object is created and do not have a return type.

Difference from method overloading:

  • Method overloading: multiple methods with the same name but different parameters; you call them explicitly.
  • Constructor overloading: multiple constructors with the same name (class name) but different parameters; called automatically when creating objects with new.

Example: Constructor Overloading

public class Car {
    String brand;
    String model;
    int year;

    // Constructor with no parameters
    public Car() {
        this("Unknown", "Unknown", 0); // calls the 3-parameter constructor
    }

    // Constructor with brand only
    public Car(String brand) {
        this(brand, "Unknown", 0); // calls the 3-parameter constructor
    }

    // Constructor with all parameters
    public Car(String brand, String model, int year) {
        this.brand = brand;
        this.model = model;
        this.year = year;
    }
}

Usage:

public class Main {
    public static void main(String[] args) {
        Car car1 = new Car();                        // Default constructor
        Car car2 = new Car("Toyota");               // Constructor with 1 parameter
        Car car3 = new Car("Tesla", "Model 3", 2023); // Constructor with 3 parameters

        System.out.println(car1.brand); // Unknown
        System.out.println(car2.brand); // Toyota
        System.out.println(car3.brand); // Tesla
    }
}

Key Points:

  • Constructor name must match the class name exactly
  • No return type (not even void)
  • Called automatically when an object is created
  • Can be overloaded with different parameters
  • this(...) can be used to call another constructor in the same class
