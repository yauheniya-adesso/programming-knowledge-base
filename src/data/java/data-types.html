In Java, all data you work with falls into two broad categories that behave differently in memory and in how they’re used:

  • Primitive types are the built‑in, low‑level data types that store simple values directly in memory and have fixed sizes.
  • Non‑primitive types are more complex structures created from classes, and they store references to objects rather than the actual data itself.
Type Description Example
Primitive Types
byte 8‑bit integer, stores −128 to 127. byte b = 100;
short 16‑bit integer, −32,768 to 32,767. short s = 30000;
int 32‑bit integer, most commonly used. int age = 25;
long 64‑bit integer, for very large numbers. long distance = 100000L;
float 32‑bit decimal, less precise than double. float pi = 3.14f;
double 64‑bit decimal, more precise. double price = 19.99;
char Single 16‑bit Unicode character. char grade = 'A';
boolean Represents true or false. boolean isJavaFun = true;
Non‑Primitive Types
String Sequence of characters; most commonly used non‑primitive type. String name = "Java";
Arrays Fixed‑size collection of elements of the same type. int[] nums = {1, 2, 3};
Object Root of all Java classes; every class inherits from Object. Object obj = new Object();
BigInteger Handles integers larger than long can store. BigInteger big = new BigInteger("123456789");
BigDecimal High‑precision decimal numbers; ideal for currency. BigDecimal price = new BigDecimal("19.99");
enum Custom type representing a fixed set of constants. enum Day { MON, TUE, WED }
List<T> Ordered collection from the Java Collections Framework. List<String> list = List.of("A", "B");
Map<K,V> Key‑value data structure. Map<String, Integer> map = Map.of("A", 1);
Set<T> Collection of unique elements. Set<Integer> set = Set.of(1, 2, 3);
var Type inference keyword (Java 10+). Compiler determines the type. var count = 10;
