Encapsulation is the bundling of data and methods that operate on that data within a class, restricting direct access to some components.

Private Fields with Getters and Setters:

public class AIModel {
    private String modelName;
    private double accuracy;
    
    // Constructor
    public AIModel(String modelName, double initialAccuracy) {
        this.modelName = modelName;
        this.accuracy = initialAccuracy;
    }
    
    // Getter for modelName
    public String getModelName() {
        return modelName;
    }
    
    // Getter for accuracy
    public double getAccuracy() {
        return accuracy;
    }
    
    // Method to improve model accuracy (like training)
    public void train(double improvement) {
        if (improvement > 0) {
            accuracy += improvement;
        }
    }
    
    // Method to reduce accuracy (e.g., due to bad data)
    public void degrade(double loss) {
        if (loss > 0 && loss <= accuracy) {
            accuracy -= loss;
        }
    }
}

Using Encapsulated Class:

public class Main {
    public static void main(String[] args) {
        AIModel model = new AIModel("GPT", 0.85);
        
        // Cannot access directly:
        // model.accuracy = 1.0; // ERROR
        
        // Use public methods instead
        model.train(0.05);
        System.out.println("Accuracy: " + model.getAccuracy()); // Output: 0.90
        
        model.degrade(0.02);
        System.out.println("Accuracy: " + model.getAccuracy()); // Output: 0.88
    }
}

Benefits of Encapsulation:

  • Data Hiding: Internal state (e.g., model accuracy) is protected from direct access
  • Validation: Control how data changes (e.g., prevent negative accuracy)
  • Flexibility: Internal logic (training, updates) can change without affecting external code
  • Security: Prevent invalid or unsafe modifications
