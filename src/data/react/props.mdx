**Props** (short for *properties*) are read-only inputs passed from a parent component to a child. They make components reusable and configurable.

<div className="flex gap-4 items-start">
  <div className="w-1/3 min-w-0">
    <p><strong>Parent — passing props:</strong></p>
    <pre className="overflow-x-auto"><code className="language-jsx">{`function App() {
  return (
    <UserCard
      name="Alice"
      age={30}
    />
  );
}`}</code></pre>
  </div>
  <div className="w-1/3 min-w-0">
    <p><strong>Child — receiving props:</strong></p>
    <pre className="overflow-x-auto"><code className="language-jsx">{`function UserCard({ name, age }) {
  return (
    <div>
      <p>Name: {name}</p>
      <p>Age: {age}</p>
    </div>
  );
}`}</code></pre>
  </div>
  <div className="w-1/3 min-w-0">
    <div className="border-1 border-blue rounded-lg overflow-hidden text-sm">
      <div className="bg-blue-50 px-4 py-2 border-b border-blue"><strong>Rendered result</strong></div>
      <div className="p-4">
        <p>Name: Alice</p>
        <p>Age: 30</p>
      </div>
    </div>
  </div>
</div>

<ul className="list-arrow">
  <li>Props flow <strong>one way</strong> — from parent to child</li>
  <li>A component must <strong>never modify</strong> its own props</li>
  <li>You can pass any value: strings, numbers, arrays, objects, or functions</li>
</ul>

---

## Default Props

If a prop is not passed by the parent, you can provide a **default value** directly in the function signature using `=`.

<div className="flex gap-4 items-start">
  <div className="w-1/3 min-w-0">
    <p><strong>Parent — passing one prop:</strong></p>
    <pre className="overflow-x-auto"><code className="language-jsx">{`function App() {
  return (
    <ModelCard provider="Anthropic" />
  );
}`}</code></pre>
  </div>
  <div className="w-1/3 min-w-0">
    <p><strong>Child — with default props:</strong></p>
    <pre className="overflow-x-auto"><code className="language-jsx">{`const ModelCard = props => (
  <div>
    <p>Provider: {props.provider}</p>
    <p>Model: {props.model}</p>
    <p>Knowledge Cutoff: {props.cutoff}</p>
  </div>
);

ModelCard.defaultProps = {
  model: 'Claude Sonnet 4.6',
  cutoff: 'Aug 2025',
};`}</code></pre>
  </div>
  <div className="w-1/3 min-w-0">
    <div className="border-1 border-blue rounded-lg overflow-hidden text-sm">
      <div className="bg-blue-50 px-4 py-2 border-b border-blue"><strong>Rendered result</strong></div>
      <div className="p-4">
        <p>Provider: Anthropic</p>
        <p>Model: Claude Sonnet 4.6</p>
        <p>Knowledge Cutoff: Aug 2025</p>
      </div>
    </div>
  </div>
</div>

<ul className="list-arrow">
  <li><code>defaultProps</code> values are used only when the prop is <code>undefined</code> (not when it is <code>null</code>)</li>
  <li>
    The <code>props =></code> syntax (used above) keeps access to the full props object — useful when you need to pass all props down or access them dynamically
    <ul className="list-dash">
      <li>
        The modern alternative is to set defaults inline using destructuring:
        <pre className="overflow-x-auto inline-block"><code className="language-jsx">{`const ModelCard = ({
  provider,
  model = 'Claude Sonnet 4.6',
  cutoff = 'Aug 2025',
}) => (
  <div>
    <p>Provider: {provider}</p>
    <p>Model: {model}</p>
    <p>Knowledge Cutoff: {cutoff}</p>
  </div>
);`}</code></pre>
      </li>
    </ul>
  </li>
</ul>
